# Contributing to SmilePnP

Thank you for your interest in contributing to SmilePnP! This document provides guidelines for contributing to the project.

## Ways to Contribute

- Report bugs
- Suggest new features
- Improve documentation
- Submit bug fixes
- Translate documentation

## Reporting Bugs

Before submitting a bug report:
1. Check the [Troubleshooting Guide](docs/troubleshooting.md)
2. Search existing [Issues](https://github.com/votre-username/SmilePnP/issues)
3. Test with the latest version

**Bug Report Template:**
```
**Describe the bug**
A clear description of what the bug is.

**To Reproduce**
Steps to reproduce the behavior:
1. Initialize scene
2. Load clip '...'
3. Click on '....'
4. See error

**Expected behavior**
What you expected to happen.

**Screenshots**
If applicable, add screenshots.

**Environment:**
- OS: [e.g., Windows 11, macOS 14.2]
- Blender Version: [e.g., 4.4, 5.0]
- SmilePnP Version: [e.g., 0.1.0]
- OpenCV Version: [from add-on preferences]

**Error Messages**
```
Paste any error messages from Blender console
```

**Additional context**
Any other relevant information.
```

## Suggesting Features

Feature suggestions are welcome! Please:
1. Check if the feature already exists
2. Search existing feature requests
3. Describe the use case clearly
4. Explain expected benefits

## Translation

To translate documentation:
1. Create `docs/lang/` folder (e.g., `docs/fr/` for French)
2. Translate markdown files
3. Maintain same structure as English docs
4. Update README.md with language links
